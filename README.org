#+TITLE: Larpi's NixOS Config
#+AUTHOR: larpi
#+OPTIONS: toc:2

Manages all of Ray's system configurations using the *Dendritic pattern* —
a thin ~flake.nix~ that hands everything off to ~import-tree~, with each
~.nix~ file under ~modules/~ being an independent flake-parts module.

Supports: macOS (nix-darwin), NixOS desktops, and standalone Linux via
Home Manager.

* Fresh install (from nothing)

** macOS / NixOS / Linux

#+begin_src bash
curl -sSfL https://raw.githubusercontent.com/kjwsl/nixos-config/main/bootstrap.sh | bash
#+end_src

The script will:
1. Install Nix via [[https://determinate.systems][Determinate Systems]] installer
2. Clone this repo to ~~/.config/nix-config~
3. Apply the right configuration for your OS automatically

** What gets applied per OS

| OS | Configuration applied |
|----+-----------------------|
| macOS | ~darwinConfigurations.macbook~ via nix-darwin |
| NixOS | ~nixosConfigurations.<hostname>~ via nixos-rebuild |
| Linux | ~homeConfigurations.linux~ via home-manager |

* Day-to-day usage

All common operations go through ~setup.nu~:

#+begin_src bash
nu setup.nu <command>
#+end_src

| Command | What it does |
|---------+--------------|
| ~system [switch\|build\|test]~ | Rebuild full system (NixOS or macOS) |
| ~home [linux\|darwin\|termux]~ | Rebuild home-manager only |
| ~update [input]~ | Update flake inputs (all, or a specific one) |
| ~gc [-d 30d]~ | Garbage collect old generations |
| ~check~ | Run ~nix flake check --no-build~ |
| ~fmt~ | Format all ~.nix~ files with alejandra |
| ~hooks~ | Install repo-local git hooks |
| ~hooks --status~ | Show global + repo-local hook status |
| ~info~ | Show current system info |

** Common workflows

#+begin_src bash
# After editing any .nix file — rebuild macOS system
nu setup.nu system

# Rebuild just home-manager (faster, no sudo)
nu setup.nu home

# Update all flake inputs then rebuild
nu setup.nu update
nu setup.nu system

# Update a single input
nu setup.nu update nixpkgs

# Garbage collect anything older than 30 days
nu setup.nu gc -d 30d

# Check the flake evaluates without errors
nu setup.nu check
#+end_src

* Adding things

** New Home Manager program/tool

Drop a ~.nix~ file under ~modules/tools/~ or ~modules/shell/~:

#+begin_src nix
{ ... }: {
  flake.modules.homeManager.base = { pkgs, ... }: {
    programs.mytool = {
      enable = true;
    };
  };
}
#+end_src

~import-tree~ picks it up automatically — no registration needed.

** New NixOS host

1. Create ~modules/hosts/<name>.nix~
2. Create ~modules/_hardware/<name>.nix~ (run ~nixos-generate-config~ on target)
3. Set ~configurations.nixos.<name>.module~ in the host file
4. Wire up home-manager: ~home-manager.users.ray = config.flake.modules.homeManager.base~

** Enable a stack on a NixOS host

In the host file:

#+begin_src nix
my.stacks.gaming.enable = true;
my.stacks.ai.enable     = true;
my.stacks.music.enable  = true;
#+end_src

* Safeguards

** Global pre-push hook (all repos)

Managed by ~modules/tools/git.nix~. After ~home-manager switch~, every new
~git clone~ or ~git init~ automatically gets a ~.git/hooks/pre-push~ that:

1. *Scans for secrets* — AWS keys, GitHub PATs, OpenAI keys, private key
   headers, literal passwords
2. *Checks nix flakes* — runs ~nix flake check --no-build~ if a ~flake.nix~
   is present in the repo root
3. *Chains to repo-local hooks* — runs ~.githooks/pre-push~ if present,
   so repos can add their own checks

For existing repos, run ~git init~ (safe, non-destructive) to pick up
the template hook.

Bypass: ~GIT_PUSH_NO_SECRET_SCAN=1 jj git push~

** Repo-local hook (nixos-config only)

~.githooks/pre-push~ evaluates the system-specific flake attribute
(~darwinConfigurations.macbook~, ~nixosConfigurations.nixos~, etc.)
to catch errors that generic ~flake check~ might miss.

Installed via: ~nu setup.nu hooks~
